# Assets Folder - Required Images

This folder should contain the following images for the PicToPDF website:

## App Icons
- `app-icon-64.png` - 64x64px app icon for navigation
- `app-icon-192.png` - 192x192px app icon for touch devices
- `favicon.ico` - 32x32px favicon for browser tabs

## Hero Section
- `phone-mockup.png` - Phone mockup showing app interface (600x800px recommended)
- `og-image.png` - Open Graph image for social sharing (1200x630px)

## How It Works Section
- `step-1.png` - Screenshot showing document capture (400x600px)
- `step-2.png` - Screenshot showing image editing (400x600px)
- `step-3.png` - Screenshot showing PDF creation (400x600px)

## Privacy Section
- `privacy-shield.svg` - Privacy/security illustration (300x300px)

## Download Section
- `indus-store-badge.png` - Indus App Store download badge
- `google-play-badge.png` - Google Play Store download badge

## Instructions to Add Real Images

1. **Extract App Icon**: Use the existing `icon2.jpeg` from the root directory
   - Resize to 64x64px and 192x192px
   - Convert to PNG format
   - Create favicon.ico from 32x32px version

2. **Create Screenshots**: Take actual app screenshots
   - Use Android emulator or real device
   - Capture key screens: home, camera, editing, PDF creation
   - Resize to consistent dimensions

3. **Phone Mockup**: 
   - Use online tools like Mockuphone.com
   - Insert real app screenshots into phone frame
   - Export as PNG with transparent background

4. **Store Badges**:
   - Download official badges from:
     - Indus App Store developer resources
     - Google Play Console brand guidelines
   - Use official colors and dimensions

5. **Privacy Illustration**:
   - Create or download shield/lock icon
   - Use SVG format for scalability
   - Match website color scheme

## Temporary Placeholders

Until real images are added, the website will show:
- Colored rectangles for missing images
- Alt text descriptions
- Graceful degradation

## Image Optimization

Before adding images:
- Compress images using TinyPNG or similar
- Use WebP format where supported
- Ensure images are under 500KB each
- Add proper alt text for accessibility

## File Naming Convention

Use lowercase with hyphens:
- `app-icon-64.png` ✅
- `AppIcon64.png` ❌
- `app_icon_64.png` ❌
